# Deploy AssetFlow

AssetFlow uses two services: a React/Vite frontend and an Express/MongoDB API.

## 1. Create a MongoDB Atlas database

Create a free MongoDB Atlas cluster and copy its connection string. In Atlas Network Access, allow the host running the Render service (or temporarily allow `0.0.0.0/0` for a demo). Store the URI only as an environment variable; do not commit it.

## 2. Deploy the API with Render

1. In Render, choose **New > Blueprint** and select this GitHub repository and the `ragasudha` branch.
2. Render reads `render.yaml` and creates the `assetflow-api` service.
3. Enter the `MONGODB_URI` value when prompted.
4. After deployment, run the Render Shell command `npm run seed` from the `server` directory, or seed the database from a trusted local environment using the same Atlas URI.
5. Confirm `<API URL>/api/health` returns `{ "status": "ok" }`.

## 3. Deploy the frontend with Vercel

1. Import this repository into Vercel, selecting the `ragasudha` branch.
2. Add the environment variable `VITE_API_URL` with the Render API URL, for example `https://assetflow-api.onrender.com`.
3. Deploy. Vercel uses `vercel.json` and serves the Vite build from `client/dist`.

## Production environment variables

| Service | Variable | Value |
| --- | --- | --- |
| Render API | `MONGODB_URI` | MongoDB Atlas connection URI |
| Render API | `JWT_SECRET` | Generated by the Render Blueprint |
| Vercel frontend | `VITE_API_URL` | Public Render API URL, no trailing slash |

Use the deployed Vercel URL for the UI. Seeded demo accounts use the password configured in `server/.env.example`.
